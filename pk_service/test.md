
知识竞赛答题
1. web利用token来获取userid
2. 发送给pk 然后利用join来进入 传入find type 选择敌人 随机 复仇
3. 根据类型来判断敌人
   1. 复仇 查对局表,id,id,time,winner; 如果全胜或者没有对局就返回错误
   2. 随机选择 利用 choose 接口
   3. 选择敌人 传入 userid
   1. 两者不要求同步，只要答完，然后对方收到请求，开始答，比较分数即可。


流程 离线打
web 答题流程
1. 发送 web 请求
2. web 向 pk_srv 发送 join
3. pk_srv 返回请求,包含题目和答案

4. 送回答案数组 利用 redis 缓存 该用户的答案 然后将比对 (设置过期时间)
5. 比对成功; 将比分送回 pk_srv 然后将比分写入 redis 
6. 然后将挑战消息发送消息队列
   1. 不应战 通知 挑战者胜利
   2. 应战 通知 填入对局 传入 挑战者id
      1. web 答题流程进行下来，送回答案数组，然后与 redis 里面的
